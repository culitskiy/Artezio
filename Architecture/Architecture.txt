В качестве примера представим веб приложение ToDo, которое позволяет пользователям ставить себе задачи.
Для этого приложения подойдет трехуровневая архитектура, так как она обеспечит производительность, надежность и безопасность.

Приложение будет состоять из следующих компонентов:
Интерфейс(он будет взаимодействовать с пользователем);
Сервер приложения(он будет выполнять биснес логику);
Сервер базы данных(он будет управлять данными и взаимодействовать с БД).
Интерфейс отрисовывает приложение и отправляет на сервер действия пользователя.
Сервер приложения выполняет бизнес логику, исходя из действий пользователя, и формирует и отправляет запросы на сервер БД.
Сервер базы данных выполняет запросы и отдает результат.

Благодаря этой архитектуре у пользователя нет доступа к базе данных, это обеспечит безопасность.
Также можно сделать проверки на всех уровнях.
Так как на стороне пользователя только отрисовка (тонкий клиент), то будет высокая производительность.
Надежности можно добиться бэкапом данных и резервными серверами.ы
